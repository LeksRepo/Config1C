Регистрация в программе дополнительных отчетов и обработок

Существуют два типа дополнительных отчетов и обработок:

  • Назначаемые - предназначены для конкретных документов или списков программы, которых может быть несколько, при этом список доступных назначений определен разработчиком заранее и включен в текст дополнительного отчета или разработки. При регистрации можно только выбрать их из списка.
  • Глобальные или общие - содержат команды, вызываемые только из разделов командного интерфейса. Команды не зависят от конкретных объектов программы. Список разделов для размещения команд определен разработчиком заранее и включен в файл дополнительного отчета или обработки, его невозможно изменить.

Какие основные сведения необходимо заполнить

  • Наименование - указывается разработчиком и выводится автоматически, при необходимости можно изменить.

  • С помощью поля Публикация можно включить или отключить доступность обработки или отчета для использования. Реквизит может принимать следующие значения:

    • используется - обработка доступна для использования;
    • режим отладки - обработка доступна для использования только пользователю с ролью "Добавление и изменение дополнительных обработок" или ролью "Полные права" (в списке отображается красным цветом);
    • отключена - обработка недоступна для использования, в списке становится неактивной. Такая возможность удобна, если нужно прекратить использование обработки на время, но не удалять ее, в этом случае все настройки сохраняются и при переводе в другой режим обработка начинает выполняться как раньше.
  • Группа - включите дополнительную обработку или отчет в одну из групп в списке. Группы помогают упорядочить дополнительные отчеты и обработки.
  • Ответственный - выберите пользователя, который будет отслеживать состояние дополнительного отчета или обработки, обновлять и т.п. По умолчанию "Администратор".
  • Комментарий - введите подробное описание.

Какие действия возможны при регистрации дополнительных отчетов и обработок

С помощью команд Загрузить из файла и Выгрузить в файл можно обновить обработку во внутреннем хранилище программы, а также выгрузить ее из хранилища в файл на диске компьютера. При выполнении команд нужно указать путь к файлу. В некоторых режимах использования программы (например, в модели сервиса) для некоторых обработок команды могут быть не доступны.

Безопасный режим работы

Поскольку дополнительные отчеты и обработки представляют собой подключаемые программные модули (разработанные на встроенном языке "1С:Предприятия"), они потенциально могут реализовывать любую функциональность, которая могла бы быть реализована в самой программе - выполнять действия, для выполнения которых недостаточно прав у текущего пользователя программы, а также взаимодействовать с внешними ресурсами (Интернет-ресурсы, стороннее ПО, установленное на сервере и т.д.), что при определенных обстоятельствах может служить причиной уязвимости в программе. Для предотвращения подобной ситуации в программе предусмотрен специальный безопасный режим подключения дополнительных отчетов и обработок - при использовании этого режима для дополнительных отчетов и обработок становится недоступным использование Интернет-ресурсов и стороннего ПО, а также выполнение операций с данными, для которых недостаточно прав доступа текущего пользователя программы. Для большинства дополнительных отчетов и обработок достаточно подключения в безопасном режиме.

В некоторых случаях дополнительные отчеты и обработки не могут функционировать в безопасном режиме - например, дополнительная обработка, реализующая получение данных с некоторого Интернет-сайта не сможет функционировать без возможности подключения к этому Интернет-сайту. В этом случае дополнительный отчет или обработка может быть подключен к программе без установки безопасного режима (это может сделать только администратор). При загрузке такого дополнительного отчета или обработки будет выведено соответствующее предупреждение - такие обработки следует загружать только из надежных источников (поставщиков) или перед загрузкой самостоятельно выполнять их аудит.

При использовании клиент-серверного режима работы программы и клиентских и серверных лицензий уровня "КОРП" возможно дополнительно ограничить перечень операций для таких обработок только явно запрошенными с помощью создания для дополнительной обработки "персонального" профиля безопасности. В этом случае, для обработки останутся недоступными все потенциально "небезопасные" операции, за исключением явно одобренных администратором (например, будет предоставлена возможность получения данных только с конкретного Интернет-сайта, и останутся недоступными другие Интернет-ресурсы, внешнее ПО и привилегированный режим).

Подробнее о безопасном режиме и профилях безопасности см. документацию по платформе 1С:Предприятие.

Особенности регистрации назначаемых дополнительных обработок и отчетов

Для дополнительных обработок или отчетов, предназначенных для работы со списками или документами программы, можно выбрать из списка объекты, в которых будут доступны команды обработки или отчета.

  • Назначение -  для выбора объектов программы нажмите гиперссылку, которая принимает значение Не определено (если ни одного объекта не выбрано) или перечисляет выбранные объекты. С помощью флажков выберите нужные объекты.
  • Показывать - команды обработок или отчетов могут быть доступны:
    • из списка выбранных объектов, если установлен флажок В списках; 
    • - из окна самого объекта, если установлен флажок В формах элементов справочников и документов. Можно установить оба флажка одновременно.
  • Команды - перечень команд заполняется по умолчанию из списка, заполненного разработчиком. Эти команды становятся доступными в указанном в назначении списке или документе при подключении дополнительного отчета или обработки.

Особенности регистрации глобальных (общих) дополнительных отчетов и обработок

Для обработок и отчетов, которые вызываются из командного интерфейса (не прикрепляются к объектам конфигурации),необходимо указать разделы интерфейса, в которых команды обработок будут доступны через команды Дополнительные отчеты и Дополнительные обработки. Перечень команд и их настройки могут размещаться на отдельной вкладке Команды.

  • Размещение - для выбора разделов нажмите гиперссылку, которая принимает значение Не опредено (если ни одного раздела не выбрано) или перечисляет выбранные разделы. С помощью флажков выберите нужные разделы программы.

Для каждой команды можно указать:

  • в колонке Быстрый доступ нажмите  и выберите из списка пользователей, для которых команда будет доступна по умолчанию. По умолчанию проставлено значение "Нет".
  • в колонке Расписание для каждой команды глобальных отчетов и обработок, исполняемых в серверном контексте, нажмите  и задайте расписание, по которому будет запускаться регламентное задание для выполнения этой команды. Расписание неприменимо для команд с вариантом запуска "Открытие форм".
